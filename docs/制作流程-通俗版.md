# 云上长征 — 制作流程（通俗版）

> 本文档用通俗易懂的语言介绍"云上长征"网站是怎么做出来的，适合没有编程背景的读者。

---

## 一、这个网站是做什么的？

简单说，"云上长征"是一个**数字博物馆**。它用网页的形式，把四川六种特产（冰糖橙、猕猴桃、豆腐、茶叶、腊肉、白酒）和它们背后的长征故事串联起来，让任何人在手机上就能"云游"这段历史。

网站支持中文和英文两种语言，有图片、文字、语音、动画地图，还有专门的二维码方便分享。

---

## 二、制作网站一共分几步？

做一个网站，就像盖一栋房子。大致分为五个阶段：

### 第一步：准备"建筑材料"（素材收集）

在动手写代码之前，我们先准备好了所有内容：

- **文字素材**：6 个产品的中英文故事、产品介绍（来自 Word 文档）
- **图片素材**：产品实拍图、产品页背景图、产地溯源地地图截图、四川地图底图
- **音频素材**：6 段英文语音导览（MP3 格式）
- **品牌素材**：Logo、配色方案（深红 + 金色 + 米白）

所有素材按类别整理好，放在对应的文件夹里，就像盖房子前先把砖瓦水泥备齐。

### 第二步：画"设计图"（页面设计）

就像盖房子需要图纸，网站也需要先设计好长什么样：

- **色彩体系**：主色是深红色（#8B1E24，象征红色文化），点缀金色（#D4AF37，象征品质感），底色是温暖的米白色
- **页面结构**：定下来一共要做哪些页面——一个首页、六个产品详情页
- **布局规划**：首页上面是品牌开场，下面是一张可以互动的地图；产品页依次是开场图、故事、产品介绍、地图、文化价值

### 第三步："盖房子"（写代码）

这一步是把设计变成真正能在浏览器里打开的网页。用到的工具叫 **Next.js**，你可以理解为一套专门用来"搭网站"的积木系统。

整个网站在电脑上的工程文件夹里，就像一个装满各种文件的工具箱。主要做的事情包括：

1. **搭建框架**：用 Next.js 把网站的"骨架"搭起来——导航栏、页面切换、底部信息栏
2. **填充内容**：把第一步准备好的文字、图片、音频逐个填进对应的页面位置
3. **做动效**：地图上的路线会动、文字会渐显、光点会呼吸闪烁——这些动画效果用的是 motion 这个工具
4. **做中英文切换**：网站里每个文字都有中英两个版本，点一下按钮就切换
5. **做音频播放**：做了一个固定在页面底部的播放器，点一下就能听语音
6. **做地图**：首页的四川地图上画了长征路线，九个地点用光点标记，点击可以弹出特产介绍
7. **做二维码**：生成六张二维码，分别对应六个产品页面，扫码就能直达

整个过程，**就像把设计图纸上的每一块区域，用代码"翻译"成浏览器能显示的网页**。

### 第四步："发布上线"（部署）

代码写好之后，网站还只是在我们的电脑上，别人看不到。需要把它"上传"到互联网上。

我们用的发布平台叫 **Netlify**，它就像一个"网页托管所"。把代码上传后，Netlify 会自动处理后续工作，然后生成一个公开网址，世界各地的人都能访问。

整个发布过程只需要在电脑上输入一行命令，等待几分钟就完成了。

### 第五步：还可以离线使用（PWA）

网站还加了一个叫 **PWA** 的功能。简单说，就是用户第一次浏览网站后，即使手机没网了，下次打开还能看到基本内容。就像把网页"下载"到了手机里。

---

## 三、用到的主要"工具"

为了方便理解，把技术名词换成生活中的比喻：

| 工具 | 比喻 | 实际作用 |
|------|------|----------|
| Next.js | 建筑框架 | 网站的"骨架"，负责页面怎么组织、怎么跳转 |
| React | 积木系统 | 把页面拆成一个个可重复使用的"组件"，像搭积木一样组合 |
| TypeScript | 纠错器 | 在写代码时自动检查错误，防止"拼写错误"导致网页崩溃 |
| Tailwind CSS | 装修材料 | 负责网站的颜色、字体、间距、圆角等所有外观样式 |
| motion | 动画师 | 负责页面里所有的动画效果（淡入、滑动、闪烁等） |
| next-intl | 翻译官 | 负责中文和英文之间的切换 |
| Netlify | 快递员 | 把做好的网站"送到"互联网上 |

## 五、总结

"云上长征"网站的制作，本质上是把**文字、图片、音频**这些素材，通过代码变成了一个**能在浏览器里交互的网页**。整个过程就像盖房子：先备料、画图纸、再施工、最后交付入住。

即使不懂编程，只要理解"素材 → 设计 → 开发 → 上线"这四个环节，就能把握整个制作流程的全貌。
