## 4. 核心实现

### 4.1 首页交互式地图

**文件**：`src/components/homepage/sichuan-illustrated-map.tsx`

- 使用 800×295 viewBox 内联 SVG
- 底层为四川地理照片 `<image>` 元素
- 河流曲线和红飘带装饰使用 `motion.g` 驱动的视差滚动（`useScroll` + `useTransform`）
- 9 个路线节点由 `ROUTE_WAYPOINTS` 数据驱动，通过 `MapHotspot` 渲染
- 邛崃节点有特殊的"灯塔"效果：多层呼吸光环 + 垂直光束 + 径向辉光

**路线路径**：`src/components/animations/long-march-route-svg.tsx`
- 基于 9 个节点坐标构建三次贝塞尔曲线
- 7 层叠加渲染：环境光晕 → 中光晕 → 主光晕 → 暗影 → 金红实线 → 深红核心 → 虚线高光 → 流动光点动画
- 流动光点使用 `stroke-dashoffset` 动画

**热点动画**：`src/components/animations/map-hotspot.tsx`
- 历史节点（瑞金、遵义）：淡红色圆点
- 产品节点：脉冲式金红圆环（3圈层叠，不同 opaticty + scale 动画周期）
- 悬停卡片：显示产品名称 + "了解详情 →" 指示

**侧边信息面板**：`src/components/homepage/story-panel.tsx`
- 使用 `motion.div` + `AnimatePresence` 实现滑入动画
- 桌面端：右侧 420px 宽面板，毛玻璃效果
- 移动端：底部弹出 sheet，拖拽手柄指示
- 按 Escape 或点击遮罩关闭

### 4.2 产品页组件编排

**文件**：`src/components/template/product-story-template.tsx`

```typescript
// 伪代码结构
export function ProductStoryTemplate({ data, messages, locale }) {
  const { register } = useAudio();
  
  useEffect(() => {
    register(data.slug, data.audio);
  }, [data.slug]);
  
  return (
    <>
      <HeroSection data={data} locale={locale} />        {/* 1. 开场 */}
      <StorySection data={data} locale={locale} />        {/* 2. 品牌故事 */}
      <ProductDetailSection data={data} locale={locale} /> {/* 3. 产品介绍 */}
      <OriginMapSection data={data} locale={locale} />    {/* 4. 产地溯源 */}
      <CulturalValueSection data={data} locale={locale} /> {/* 5. 文化价值 */}
      <AudioPlayer />
    </>
  );
}
```

### 4.3 音频系统

**架构**：React Context 模式，全局单一 `Audio` 元素

**文件**：
- `src/components/audio/audio-context.tsx` — Context Provider，管理 `Audio` 实例
- `src/components/audio/audio-player.tsx` — 固定底部播放条 UI
- `src/components/audio/audio-hero-status.tsx` — 产品页开场区的播放触发按钮
- `src/components/audio/audio-visualizer.tsx` — Canvas 波形可视化（装饰性）

**交互逻辑**：
1. 进入产品页 → `register(slug, audioPath)` 注册音频源
2. 点击 AudioHeroStatus → `play(slug)` 播放
3. AudioPlayer 显示进度条+时间，可拖动 seek
4. 切换产品 → 自动切换音频源
5. 全局只有一个 `Audio` DOM 元素，通过 context 共享

**音频文件**：6 段英文产品导览，MP3 格式，时长 215-225 秒

### 4.4 国际化

**方案**：`next-intl` 基于文件的消息系统，`localePrefix: 'always'`

```
messages/
├── zh.json    # 中文文案（common, homepage, product, audio, footer）
└── en.json    # 英文文案

src/i18n/
├── routing.ts   # createNavigation({ locales: ['zh','en'], localePrefix: 'always' })
└── request.ts   # getRequestConfig → loadMessages(locale)

src/middleware.ts # createMiddleware(routing) → matcher: ['/', '/(zh|en)/:path*']
```

产品页双语内容（故事、产品介绍、文化价值）直接存储在 `src/data/{slug}.ts` 的数据结构中，通过 `locale` prop 选择。

### 4.5 动画系统

所有动画组件基于 `motion`（原 framer-motion v12）。

| 组件 | 动画类型 | 参数 |
|------|----------|------|
| `FadeInView` | 进入视口渐显 | 方向、延迟、时长、`viewport.once` |
| `TextReveal` | 文字裁切展开 | `clipPath: polygon(0 0, 0 0, ...)` → `polygon(0 0, 100% 0, ...)` |
| `PageTransition` | 路由切换过渡 | opacity + scale，0.4s |
| `ParallaxContainer` | 视差滚动 | `useScroll` + `useTransform`，-30px ~ +30px |
| `MapHotspot` | 无限脉冲 | 多层圆环 opacity + scale 循环 |
| `LongMarchRouteSVG` | 流动光点 | `stroke-dashoffset` 线性循环 |
| `MountainSilhouette` | 累计视差 | 4 层 SVG 不同速度移动 |
| `GoldParticles` | Canvas 粒子 | 18 粒，随机初始位置+漂移+淡入淡出 |
| `ScrollDownIndicator` | 脉冲竖线 | `scaleY` 循环 |

无障碍考虑：所有动画组件通过 `useReducedMotion()` 检测用户偏好，`prefers-reduced-motion` 时禁用动画。

### 4.6 溯源地地图

**最终方案**：静态 PNG 图片

经历了三次迭代：
1. **react-leaflet + Leaflet** → SSR 报错（`window` is not defined）
2. **OpenStreetMap iframe embed** → 外部依赖，加载不稳定
3. **静态 PNG 截图**（当前方案）→ 按 `slug` 加载 `/images/origin-maps/{slug}.png`，无外部依赖，加载可预测

### 4.7 PWA / Service Worker

**文件**：`public/sw.js`

策略：
- `CACHE_NAME: 'cloud-long-march-v2'`
- `PRECACHE_URLS: ['/offline.html']`
- Navigate 请求：**network-only**，不缓存 HTML（避免出现陈旧页面）
- 静态资源（js/css/png/webp/svg/mp3）：runtime 缓存，stale-while-revalidate
- 离线时导航请求回退到 `offline.html`

**Manifest**：`public/manifest.json`
- `display: standalone`, `theme_color: #8B1E24`, `start_url: /zh`
- 8 种尺寸图标（72-512px）

## 5. 数据模型

```typescript
interface ProductData {
  id: string;          // 'orange' | 'kiwi' | 'tofu' | 'tea' | 'pork' | 'liquor'
  slug: string;
  chineseName: string;
  englishName: string;
  sloganCN: string;
  sloganEN: string;
  heroImage: string;
  audio: string;
  audioDuration: number;
  storyCN: string[];              // 品牌故事（3-4段）
  storyEN: string[];
  productCN: string[];            // 产品描述（2段）
  productEN: string[];
  location: { regionCN: string; regionEN: string };
  coordinates: { x: number; y: number };        // SVG 地图百分比坐标
  geoCoordinates: { lat: number; lng: number };  // 真实经纬度
  culturalValuesCN: Array<{ keyword: string; description: string }>;
  culturalValuesEN: Array<{ keyword: string; description: string }>;
  seoTitleCN: string;
  seoTitleEN: string;
  seoDescriptionCN: string;
  seoDescriptionEN: string;
}
```

6 个产品数据文件的 `storyCN/storyEN` 内容来源于参考文档 `中英对照版-川蜀红色特产与长征故事（6个）2026.6.2.docx`。

## 6. 路线节点数据

**文件**：`src/data/route-waypoints.ts`

9 个路线节点，坐标经真实四川地理方位校准（y=0 为北）。节点顺序反映长征路线：瑞金 → 遵义 → 苍溪 → 剑门关 → 泸州 → 邛崃 → 雅安 → 阿坝 → 甘孜。

前两个为纯历史节点（无产品），后七个关联具体产品。`PRODUCT_POSITIONS` 提供独立的产品热点坐标，用于在 SVG 地图上精确定位。

## 8. 部署架构

### Netlify 配置

**`netlify.toml`**：
```toml
[build]
  command = "npx next build"
  publish = ".next"

[[plugins]]
  package = "@netlify/plugin-nextjs"
```

### 构建配置

**`next.config.ts`**：
- `images.unoptimized: true`（静态导出兼容）
- 自定义 `Cache-Control` 响应头：
  - HTML 路由：`no-store`（不缓存）
  - `/audio/` 和 `/images/`：`public, max-age=31536000, immutable`（1年强缓存）
  - `/sw.js`：`no-cache`（每次验证）

### 构建产物

- 12 个产品页（6×2 locale）+ 2 个首页 + sitemap + robots = 约 18 个静态页面
- 构建时间约 22 秒（TypeScript 12s + 页面生成 1s）
- 输出：`.next` 目录，由 Netlify plugin 处理部署

## 9. SEO 策略

- **静态站点地图**：`/sitemap.xml` 包含所有 locale+页面组合
- **动态元数据**：每个产品页通过 `generateMetadata()` 生成独立的 `<title>`、`<meta description>`、Open Graph、Twitter Card
- **PWA Manifest**：包含 `name`, `short_name`, `description`, `start_url`, `icons`
- **语义化 HTML**：`<h1>`-`<h3>` 层级、`role`/`aria-label` 属性
